# Sources

Upstream data snapshots used by `build/build_dict.py`. Anything here is
CC-BY-SA 4.0 — see the root `LICENSE-DATA`.

## Files

| File | Origin | Notes |
|---|---|---|
| `chars-hkilang.csv` | [hkilang/TTS `src/res/chars.csv`](https://github.com/hkilang/TTS) | Single-character Hagfa Pinyim readings. Generated by `python build/fetch_sources.py`. Not committed by default — CI fetches it on every run. |
| `flashcards-huiyang.csv` | Hakka-flashcards-site (`Hakka Vocabulary.csv`) | Phrase-level entries; four columns `普通中文,客家汉字,Hakka Pronunciation,English Definition`. Committed because it's small and hand-curated. |
| `phrases/<slug>/*.csv` | Community contributions | Same format as flashcards. Optional. |

## Refreshing hkilang data

```bash
python build/fetch_sources.py                # latest `main`
python build/fetch_sources.py --commit <sha> # pin a specific upstream commit
```

The raw upstream file is cached in `.cache/` for diff/attribution purposes
(ignored by git).

## Attribution

- **hkilang/TTS** — community-maintained Hakka TTS dataset. The right-hand
  column of the upstream `chars.csv` is the Hagfa Pinyim reading and is
  what we consume. Repo: https://github.com/hkilang/TTS.
- **Hakka-flashcards-site** — phrase curation by the flashcards project.

If you add a new source, document it here with the upstream URL, the
